# iWC 架构设计

## 设计思想

**小工具路线**：代码简单、逻辑清晰、易于维护。

- 单进程架构，无复杂状态管理
- 每条消息独立进程，不共享内存
- session 通过文件系统恢复，无数据库依赖
- 核心代码约 500 行

## 系统概览

```
微信 (ilink 长轮询) ──→ weixin.Platform ──→ engine ──→ claude.Agent ──→ Claude Code
                      ←── 回复消息       ←──         ←── stdout
```

消息处理流程：

1. Platform 长轮询接收微信消息
2. Engine 按用户路由，获取/创建 session
3. Agent 调用 `claude --print [--resume <id>]`
4. 回复分块发回微信

## 核心组件

### 1. weixin.Platform

微信消息轮询。

**职责：**
- 长轮询 `GET /ilink/bot/get_updates`
- 消息去重（messageID + createTime）
- 用户白名单过滤（allow_from）
- 持久化游标和 contextToken
- 打字状态管理

**持久化文件：**

```
~/.iwc/wechat/
├── get_updates.buf       # 轮询游标，重启后继续
├── context_tokens.json   # 用户 -> contextToken
└── sessions.json         # 用户 -> sessionID
```

### 2. engine.Engine

消息路由和业务编排。

**职责：**
- 管理每用户的 sessionID 映射
- 持久化 sessions.json
- 调用 agent.SendWithSession
- 消息分块发送（3800 字符/块）
- 控制"正在输入"状态

### 3. claude.Agent

Claude 进程管理。

**职责：**
- 执行 `claude <text> --print [--resume <id>]`
- 从 `~/.claude/sessions/` 获取新 sessionID
- 每次消息独立进程，无共享状态

**关键方法：**

```go
// 发送消息，返回回复和新的 sessionID
SendWithSession(ctx, text, sessionID) (string, string, error)

// 从 ~/.claude/sessions/ 找最新 session
findLatestSession() string
```

### 4. main / CLI

入口仅 4 个命令，无复杂参数。

```
iwc start      # 启动服务
iwc stop       # 停止服务
iwc restart    # 重启
iwc wechat setup  # 扫码配置
iwc autostart on/off  # 开机自启
```

## Session 管理

```
首次消息（无 session）:
  claude <text> --print
  → 创建新 session
  → 找最新 session 文件，提取 sessionId
  → 保存到 sessions.json[userID]

后续消息（有 session）:
  claude <text> --print --resume <sessionID>
  → 恢复该 session 对话
```

**为什么不用持久进程？**

Claude Code 的 `--print` 模式不支持长连接 stdin/stdout，无法维持持久进程。每次独立进程 + session resume 是更简单的方案。

## 配置

```toml
[wechat]
token          # ilink Bearer token
base_url       # 默认 https://ilinkai.weixin.qq.com
allow_from     # 允许的用户 ID（逗号分隔）
long_poll_timeout_ms = 35000

[claude]
cli_path = "claude"        # 或完整路径
work_dir = "/Users/admin"  # Claude 工作目录

[system]
data_dir = "/Users/admin/.iwc"  # 持久化目录
```

## 代码结构

```

├── main.go          # CLI 入口 (~250行)
├── config/config.go # 配置加载 (~100行)
├── weixin/
│   ├── types.go     # 协议类型 (~130行)
│   ├── client.go    # HTTP 客户端 (~200行)
│   └── platform.go  # 轮询 + 状态 (~350行)

└── claude/agent.go     # 进程管理 (~130行)
```

总计约 1200 行，无重型外部依赖。

## 错误处理原则

- 打字状态失败 → 记录日志，继续（不影响主流程）
- 无 contextToken → 返回提示消息
- Session 无效 → 自动创建新 session
- Claude 超时 → 返回超时提示

所有错误都有明确日志，便于排查。
