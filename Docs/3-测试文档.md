# iWC 测试文档

## 测试原则

- 手动测试为主，单元测试覆盖关键路径
- 不追求覆盖率，只追求"改动后能跑通"
- 日志清晰，错误可追溯

## 单元测试

### weixin 包

| 测试 | 说明 |
|------|------|
| `TestClientGetTypingTicket` | GetTypingTicket API，mock HTTP server |
| `TestClientSendTyping` | SendTyping API，mock HTTP server |

## 手动测试用例

### TC1: 首次消息

1. 清空 sessions.json
2. 发送"你好"
3. 验证 Claude 回复
4. 检查 sessions.json 有新记录

### TC2: 多轮对话

1. 发送第一条消息
2. 发送第二条"接着上文"
3. 验证 Claude 知道上下文

### TC3: 重启恢复

1. 发送消息，建立 session
2. `iwc stop && iwc start`
3. 发送消息，验证恢复同一 session

### TC4: 打字状态

1. 发送需要几秒处理的消息
2. 观察微信是否显示"正在输入中"
3. 验证完成后停止

### TC5: 持久化

1. 发送消息，建立 contextToken
2. 重启服务
3. 发送消息，验证 token 有效

## 运行测试

```bash
go test ./...
go test ./weixin/... -v
go test ./claude/... -v
```

## 日志

```
/tmp/iwc_stdout.log  # stdout
/tmp/iwc_stderr.log   # 日志（带时间戳）
```

查看实时日志：

```bash
tail -f /tmp/iwc_stderr.log
```

日志关键标识：

| 标识 | 含义 |
|------|------|
| 📩 收到微信消息 | 收到用户消息 |
| 📤 启动 Claude | 正在调用 Claude |
| ✅ Claude 回复完成 | 处理成功 |
| ⏰ Claude Code 响应超时 | 超过 120s |
| ⚠️ 发送正在输入状态失败 | 打字状态 API 失败（不影响主流程）|
